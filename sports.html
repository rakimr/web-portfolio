NBA Player Performance üèÄ


In this project I've conjured up a dynamic projection workbook that, ultimately, projects NBA Player performance using a plethora of formulas.
This workbook allows all of my NBA Fantasy clients to make data-driven decisions when creating their fantasy lineups.
Being that it's not basketball season at the moment, my live workbook isn't able to update properly.
So for example purposes, I'll be using an old projection workbook to display here on the webpage.
Below the projection workbook you can find detailed descriptions about each sheet.
The workbook is interactive, so you can click on the cells to view their contents.
If you'd like to inspect all formulas, just use the shortcut CTRL + ~

Per_Pos

On the Per_Pos sheet, I use the =IMPORTHTML("url", "table", 0) formula to index a table containing each NBA players Per 100 Possessions stats from basketballreference.com.
Using these stats are a much more predictive approach than using Per Game. Essentially, they are the same stats as Per Game stats except instead of taking a player's totals and dividing them by games, you take their totals and divide it by the amount of possessions.
Since im importing the basketballreference's url, this sheet updates whenever basketballreference updates.
Scrolling all the way to the right of the sheet, you will see a column labeled "Average Fantasy Points Per 100 Pos".
The values in this column are calculated using the paramaters in DraftKings scoring formula:

Point +1
Made 3pt Shot +0.5
Rebound +1.25
Assist +1.5
Steal +2
Block +2
Turnover -0.5

These parameters are multiplied by each players' respective stat.
The formula reads:(AC2*1) +(L2*0.5)+(W2*1.25)+(X2*1.5)+(Y2*2)+(Z2*2)+(AA2*-0.5)

Salaries

On the Salaries sheet, I have player fantasy data that I downloaded from the DraftKings website in the form of a csv file. I then upload the csv file onto the sheet.
Here I am able to index the players' respective fantasy salaries, and fantasy positions. Later, these categories will help evaluate the cost-effectiveness of each player.

Possessions

On the Possessions sheet, I use the =IMPORTHTML("url", "table", 0) formula to index a table containing each NBA teams total Possessions Per Game from teamrankings.com.
Since im importing the teamrankings's url, this sheet updates whenever teamrankings updates.

Minutes

On the Minutes sheet, I use the =IMPORTHTML("url", "table", 0) formula to index a table containing each NBA player's projected minutes for the day from numberfire.com.
Projected Minutes is an important aspect of an NBA players game. No matter how good your favorite player is, at the very least they will not reach their average performance if they are on limited minutes.
numberfire.com does a fantastic job at projecting how many minutes a player will play on any given night.
Since im importing the numberfire's url, this sheet updates whenever numberfire updates.

Analysis

On the Analysis sheet, I am computing predictive statistics using values indexed from the previous sheets listed above.
For the columns "Position", "Salary", "Proj Minutes", "Team", "Opponent", "Team Possessions", "Opponent Possessions", and "Avg FP Points Per 100 Pos", I am using the =INDEX(reference, [row], [column]) formula to index values from my previous sheets.
In my opinion, this is a great alternative to the =VLOOKUP(search_key, range, index, [is_sorted]) function as vlookup is not only a limited function, but the criterions can also be a bit confusing.
For the "Proj Player Possessions" values, I am projecting each players possessions for the day. I do this by first dividing each players projected minutes by 48 as there are roughly 48 total minutes in a single game. I then multiply this result by the average of each players team and opponent possessions. That calculation takes the form of: =(D3/48)*(AVERAGE(G3:H3))
Moving on to the "Proj FP" values. This column holds each players projected fantasy points for the day. These are calculated by first dividing each players projected player possessions by 100. This is because there are roughly 100 possessions in a single game. I then multiply this by each players average fantasy points per 100 possessions. The formula comes out as:=(J2/100)*I2
Last but not least is the "Cost Effectiveness" column. These values allow my clients to determine which players are the most valuable for the day.
These values are calculated by taking each players salary and dividing them by a bases of 1000, since every salary is in the thousands. That result then gets divided further by each players projected fantasy points. Here's the formula: =K2/(C2/1000)
They are then conditionally formatted to give the values a more intuitive feel.
